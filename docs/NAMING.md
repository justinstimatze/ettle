# ettle — naming

## The name

**ettle** — a Scots / Northern-English dialect verb: *to intend, to aim at, to plan or prepare ahead; to purpose.*

It names the active thing the system does — intending and preparing ahead on the team's behalf — rather than the passive thing (holding shared state). It is terse, a real word, obscure, and texturally at home with the rest of the set (kith, fettle, settle). It names a stance, not a state: a hunch is a hint you happen to have; to ettle is to act on intent ahead of time — the multiplayer, actionable counterpart to merely sensing it.

## Collision check (June 2026)

Targeted web search found no software product, package, or company named `ettle`. The namespace is clean. (`github.com/ettle`, npm, and PyPI should be confirmed at registration time, but no prominent prior use surfaced.)

## Alternatives considered and why they lost

- **murmuration** — semantically perfect: a starling flock is leaderless, collisionless, surprise-minimizing coordination, and the scale-free-correlation physics is the thesis itself. Rejected on the project's own collision rule: Murmuration Labs, Murmuration (US civic-data nonprofit, prominent), Murmuration SAS (geo-AI), plus npm `murmuration`, npm `murmurate`, and PyPI `murmuration`. Crowded across both company and package namespaces. It survives as the right *metaphor* even though it failed as the name.
- **skein** — flock-in-flight plus interwoven threads, a strong image. Rejected: **Skein** is a well-known cryptographic hash (Schneier et al., SHA-3 finalist); a dev-space collision.
- **rede** — archaic English for counsel / interpretation, register-perfect. Rejected: "rede" is Portuguese for "network" and a major Brazilian payments brand (Itaú's Rede).
- **inwit** — Middle English for inner knowledge / conscience; a clean knowledge-word in the same sensing/knowing family. Viable alternate; lost to ettle because ettle names the *active* (intend-ahead) function rather than the passive (inner-knowing) one.
- **forewit** — archaic "foresight." The project-name is free, but there is an active GitHub user `forewit` (handle taken). Minor, but enough to drop it behind ettle.
