# ettle — HANDOFF

Design stage, with a runnable PoC. Motto: **no meetings** — honestly, the sync meeting dies and the decision meeting gets shorter (see *The premise, stated honestly* below); not literal abolition. This file is the spine: what ettle is, where it stands, and the critical path. Read [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) for the flow diagram (start there — the shape is unintuitive), [docs/CONCEPT.md](docs/CONCEPT.md) for the model, [docs/N1_WEDGE.md](docs/N1_WEDGE.md) for the first thing to build, [docs/TEAM_SIM.md](docs/TEAM_SIM.md) for the multiplayer payoff demo, [docs/HORIZON.md](docs/HORIZON.md) for the extrapolated end-state (the vision, and its shadow), [docs/COMMONS.md](docs/COMMONS.md) for the Ostrom commons-governance frame, and [docs/PRIOR_ART.md](docs/PRIOR_ART.md) for the landscape.

## One-paragraph statement

A small, fully-remote, high-trust team where everyone already thinks through their work with an AI agent leaves most of its coordination latent in those sessions. The sessions already hold — or can extrapolate — what each person is doing, what they depend on, and what they are about to change. ettle makes that latent state mutually legible without meetings: each agent models its own human, keeps a directed model of each teammate, and a collective layer reconciles those models and emits only the deltas that reduce surprise. Done with a calibration loop and a privacy boundary, the team coordinates ahead of the planning cycle. Done without them, it becomes a fast, confident, unauditable misread of everyone — which is the thing to avoid.

## The premise, stated honestly

The bottleneck in such a team is humans telling other humans things their agents already know. A meeting does several jobs, though, and only one of them is information sync:

1. **Information sync** — "here's what I changed / am blocked on / you should know." This is the redundant part. Agents can collapse it to near-zero.
2. **Preference aggregation, commitment, conflict surfacing** — whose priority wins, who is now on the hook, where a latent disagreement becomes visible. This is not information transfer; it is politics among people with divergent interests, and it does not vanish because the agents are well-informed.

So the realistic claim is not "no meetings." It is: the sync meeting dies, the decision meeting gets shorter and sharper because nobody arrives uninformed. The bottleneck relocates to (a) intent not yet expressed in any artifact and (b) commitment, which is a speech-act, not an inference. ettle shrinks both but does not abolish them.

## The lever

The signal source is the **reasoning in progress**, not the artifacts after the fact. Files, email, and calendars expose intent only once it is already a commit or a sent message — too late, and the confabulation-prone version where the agent guesses why. People thinking a decision through with an agent are expressing the intent *while it is still forming*. That is the pivot-on-a-walk, except it is in a transcript and can emit a signal. This is what makes ettle tractable where ambient file-watching is not.

This closes the critical-path half — **act on the model, then check whether the action was right** — now has a real judgment signal. (The eval detail and the thread it ran on are kept local and uncommitted — derived from a real internal thread, anonymized.)
  - **Concept demos (simulations, not the product):** a single-user wedge loop (prior-decision guard + did-it-help; first run 0 false interrupts, 0 missed of the seeded scenarios) and a multiplayer "no meetings" sim (agents standing in for the humans: L1 emit → L3 knot-detection → bind-vs-surface routing; dissolves the seeded collisions ahead, standup cancelled). These illustrate the payoff shape on cheap models; they are not something a real team uses.
  - **gemot deliberation proven end-to-end locally** ([gemot](https://github.com/justinstimatze/gemot): crux extraction with a controversy score for the bind-vs-surface router, a binding compromise, EigenTrust reputation as the Ostrom sanction). ## Critical path

The swarm is **not** blocked on the multiplayer machinery — the transport is decided. A **NATS** bus carries typed atoms between machines (TLS + auth native, pub/sub, JetStream replay, self-hosted single Go binary — secure *when configured*, with the auth/TLS primitives present natively, unlike a plain MCP-over-HTTP endpoint; the ettle client cannot verify server-side enforcement, so "secure by default" is the deployment's job, not a guarantee ettle checks). **Note the substrate is a shared stream:** any team credential that can publish can also replay every teammate's atoms — the `--me` filter is presentation, not per-atom access control. **gemot** handles the crux/deliberation step (positions → crux → binding compromise + EigenTrust reputation), reached behind a negotiation seam. The human-legible side needs **no shared channel**: each person's own agent surfaces the relevant knot back to them in-session, when helpful — which also keeps the privacy boundary tight (you only ever see what your own agent judged relevant to you). The remaining protocol/privacy substrate (A2A, the privacy-preserving-MAS literature, confidential computing) is being built for us by others. ettle is blocked instead on the one thing the single-user layer never did even for a single user:

**Act on the model, and check whether the action was right.**

So the critical path is the **N=1 wedge** before anything multiplayer. The wedge is now specced — see [docs/N1_WEDGE.md](docs/N1_WEDGE.md). In brief:

1. **The behavior: the prior-decision guard.** When the live turn re-opens something the user previously closed, the session surfaces that once — as a question, before proceeding. Chosen over the pre-read and ownership-prompt candidates because it models the *person* (their past decisions), is genuinely useful at N=1, and its did-it-help signal is the same object the multiplayer calibration loop needs. Built on the single-user layer's existing closed-decision cells + hooks.
2. **Did-it-help: guard precision.** The user's next reply is classified `helped` / `overridden` / `false_interrupt`. Precision = helped / (helped + false_interrupt); `overridden` retires the cell (the loop closing, not a miss); `false_interrupt` is penalized hard via a hard absolute guard. This is the N=1 instance of CONCEPT.md's L2-vs-L1 divergence metric.
3. **Then go multiplayer.** L2 directed models + L3 merged horizon over the NATS atom bus. The conflict-detection at L3 is the actual product value — and it reuses the wedge's `GuardEvent` atom verbatim, with `trigger` pointing at another participant's closed cell.
4. **Calibration loop is mandatory at the swarm tier** — the longitudinal calibration metric scoring L2-vs-reality divergence over time. Speed amplifies miscalibration; this is what keeps "modeling people ahead, at high speed" useful rather than dangerous. **Stated plainly: this loop is unbuilt.** The fast people-modeling half (the detector) runs today; the correction half (this loop) does not — so until it exists and is shown to fail loud on divergence, ettle is the drift-prone version by default, and every safety claim that leans on calibration is borrowing against unbuilt code. The Phase-3 adjudication harness is the first running piece of it.

## Design invariants (do not trade away)

- **Calibration before speed.** A model of a person must stay correctable by that person. (See SF_LINEAGE.md for why this is the line between the good and bad versions.)
- **Contextual privacy boundary.** Each person controls what crosses their boundary. Distilling typed atoms rather than streaming transcripts is the cheap form of this; confidential computing / TEEs are the substrate at larger scale.
- **Humans remain the deciders, not just the modeled.**
- **Friction in the right spots.** Not frictionless — frictionless is impossible and undesirable where stakeholders genuinely diverge (Rakova). Remove friction from coordination/information-sync (the bullshit-meeting toil → zero); keep it at the cruxes — the genuine values choices a person should own, pre-staged as a clean either/or, never auto-decided by the mesh. The felt goal: empowered and free of bullshit meetings, but still getting the benefit of having had a great meeting, because the mesh held it on everyone's behalf. (See TEAM_SIM.md; gemot's crux-detection is the mechanism.)
- **The coordination commons is governed, not assumed.** Coordinated quality without wasted time is a common-pool resource (shared attention, legibility, trust) with a real overgrazing failure mode (every agent over-emitting). Ostrom's eight principles govern it: emit-gate as appropriation rule, calibration as monitoring, **gemot reputation as graduated sanctions**, consent-first adoption as boundaries/right-to-organize, L1/L2/L3 + the N-spectrum as polycentric nesting. The rule-making stays human; the agents are the monitored appropriators. (See COMMONS.md.)
- **No machine-speed feedback loop (runs at the speed of human decisions, not the bus).** The continuous version must not let atoms-on-the-bus burn everyone's tokens trying to keep up with each other. Non-negotiable mechanics: **L3 emits knots, never atoms** (humans are the only atom source — breaks the cascade structurally); **emit is surprise-gated** and decided by a cheap local diff before any model call; **reconcile is O(1) per tick, shared and broadcast** — not O(M) per-agent re-reconcile; debounced, content-hash-skipped, cheaply pre-filtered; per-agent token budget as the graduated sanction. The live "production hook path" is gated on these. (See SCALING.md.)
- **Consent-first, anti-viral adoption.** The unit of adoption is a team that opts in together. Never represent or contact a non-participant. See ADOPTION.md — this is a hard requirement, not a growth-tactic preference.
- **Useful at N=1.** Value must not be hostage to network effects, or adoption pressure becomes coercive.
- **Truncate the recursion.** Model self + model-of-other (depth 2), selective depth 3 for a few high-stakes pairs. Do not build the infinite metaperspective regress.

**Recently built (engine + driver + tests, on top of the PoC):**
- **Calibration harness** (`internal/eval`, `ettle eval`, Phase 3) — answers the sharpest panel critique (the "~0.6 precision" was unfalsifiable: gitignored denominator, one private thread, circular rater). Now the ground truth is a **committed synthetic corpus** (`testdata/eval/*.json`) a stranger can read; `ettle eval` scores precision (FIRM assertion quality) and recall (FIRM-or-soft coverage) against it; `--ab` runs the honest single-shot-vs-voted comparison with a **McNemar** significance test. The matcher uses an overlap-count (a Jaccard ratio wrongly penalizes verbose explanations). **Early finding:** on the small corpora voting does *not* demonstrably beat single-shot (it sometimes drops a flaky-but-real knot), and McNemar correctly refuses to claim significance at this N — which is exactly the honest, non-overclaiming result. Also fixed the voting clustering itself: `SameKnot` now uses a Jaccard threshold (was: any one shared keyword → over-merged on a hub person + common noun) and `voteKnots` uses order-invariant union-find (was: order-dependent first-match).
- **L1 live-session capture** (`internal/capture`, `ettle capture`, Phase 7) — the load-bearing piece. ettle's thesis is "model people from reasoning-in-progress, not after-the-fact artifacts" — but the public tool used to read only hand-written notes (an artifact), which the adversarial panel correctly called the wrong lever. Capture distills a real **Claude Code session transcript** into the note-equivalent digest (stated intent from the person's prompts + work committed via Edit/Write/Bash; Read/Grep exploration and subagent sidechains skipped). `ettle standup session.jsonl` now runs the whole pipeline on live reasoning. Live-validated: two synthetic session transcripts (`testdata/sessions/`) surfaced their latent auth-contract collision as a FIRM collision + decision-rights crux. The digest stays local; only distilled atoms cross. (A richer private predecessor still offers deeper L1 telemetry; this is the minimal public path that makes the repo deliver the thesis end-to-end.)
- **Multi-sample voting** — `ReconcileVoted(ctx, atoms, samples)` runs the reconcile passes N times and keeps only knots recurring across a strict majority; surviving knots carry `Votes`/`Samples` (a robustness signal kept separate from Confidence). Wired as `--samples K` (default 1 = original cost). Clustering uses a shared `SameKnot` matcher (party overlap + salient-keyword overlap, deliberately NOT Kind-keyed, since the detector relabels the same problem run to run). **Honest scope:** voting reduces run-to-run *paraphrase variance* (test-retest reliability), not *bias* — N samples of one model at one temperature aren't independent, so a systematic misread can recur to a spurious "3/3" while a rare true positive drops below the majority. It is a stability signal, not a precision fix; the Phase-3 `ettle eval --ab` harness now tests this directly and finds **no significant improvement** on the current corpora (too few discordant cases to claim one). Do not read "seen in 3/3" as "verified true."
- **Self-assumption knots (useful at N=1)** — `ReconcileSelf` surfaces a single person's own stale assumption (an earlier assumption their later atoms made false); deduped against the cross-person knots via `DedupeSelf`/`SameKnot`. Honors the "useful at N=1" invariant the pairwise/team passes structurally couldn't.
